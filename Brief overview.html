I want to share a bit my idea too. It’s a voice assistant, not too different from the current alternatives, but with a few key features:

- The motto is “A human assistant. Built for humans”. I focus on making it personal, intuitive, focused on the user, and like a real personal assistant.
- Assistants talk to each other (e.g. my assistant talks to my friends’ assistants)
- The assistant is connected to Freebase and Wolfram Alpha, so it can answer lots of general knowledge questions
- The assistant learns about the user: their preferences, friends, behaviour, etc. Then it can predict things. e.g.:
     - it learns that the user sends a “good night” message to someone every evening, the assistant will ask the user whether they want the assistant to send it instead (automatically); though I don’t think this’d be a common use case
     - it learns what pizza the user likes. When they ask the assistant “Order my favourite pizza” the assistant will show a card with the order (what type of pizza, how many etc.) and will ask “Send order of $10.99?”

Right now I’m working alone. I have in place:
1) an Android app with on-boarding screens, speech recognition, text to speech. I’m working on authentication and server communication right now
2) an Akka server (actor based, highly scalable, Continuous Delivery is almost finished). Authentication logic is done; I have some basic text processing (NLP) and input processing in place too
